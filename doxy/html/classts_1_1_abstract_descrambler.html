◆ getBitrate()

virtual BitRate ts::AbstractDescrambler::getBitrate ( )
inlineoverridevirtual

Get the plugin bitrate.

The main application may invoke getBitrate() at any time. The semantics depends on the capability.

  • Input plugin: Returns the current input bitrate of the device. Useful for real-time devices only.
  • Output plugin: Returns the current output bitrate which is used by the device.
  • Packet processing plugin: Return the current bitrate at the output of the packet processor. This can be used by packet processors which influence the bitrate by removing packets or introducing delays.

Optionally implemented by subclasses. By default, return that the plugin is not aware of the bitrate.

Returns
Plugin bitrate in bits/second. Shall return 0 on error or unknown bitrate.

Reimplemented from ts::Plugin.

◆ processPacket()

virtual Status ts::AbstractDescrambler::processPacket ( TSPacket &  pkt,
bool &  flush,
bool &  bitrate_changed 
)
overridevirtual

Packet processing interface.

The main application invokes processPacket() to let the shared library process one TS packet.

Dropping packets affect the output bitrate if the output device is a real-time one. With such devices, it is better to replace the undesired packet with a null packet.

Dropping a packet or changing its PID (including replacing a packet with a null one) affects the continuity counters of the other packets of the original PID.

Parameters
[in,out]pktThe TS packet to process.
[in,out]flushInitially set to false. If the method sets flush to true, the packet and all previously processed and buffered packets should be passed to the next processor as soon as possible.
[in,out]bitrate_changedInitially set to false. If the method sets bitrate_changed to true, tsp should call the getBitrate() callback as soon as possible.
Returns
The processing status.

Implements ts::ProcessorPlugin.
